# MindAll - AI Business Operating System (BOS)

MindAll is a production-grade AI Entrepreneur Copilot designed to act as a strategic co-founder. It leverages multi-source RAG, hardened adversarial defense, and a backend-authority tool execution model to provide grounded, executive-level business intelligence.

## 🚀 The Vision
MindAll is not a chatbot. it is a **Strategic Intelligence Engine** that helps founders navigate the "Valley of Death" by grounding AI advice in proprietary business data and real-time market reality.

## 🛡️ Level 3.5 Intelligence (Current State)
MindAll has evolved through four major strategic levels:
*   **Level 1: Deep Data Moat** — Knowledge Vault for PDF/Markdown ingestion and semantic memory.
*   **Level 2: Proactive Agency** — Daily Executive Briefings generated by background agents.
*   **Level 3: Real-Time Intelligence** — Automated strategic routing between internal memory and live web retrieval (Tavily).
*   **Level 3.5: Strategic Ledger** — Interactive memory alignment, knowledge lineage tracking, and real-time strategic context sidebars.

---

## ✨ Core Features

### 🧠 Strategic Ledger & Decision Support
*   **Strategic Context Sidebar:** Real-time visibility of Confirmed Strategy and Emerging Observations during chat sessions.
*   **Interactive Alignment:** Founders can verify, purge, or weight AI-extracted insights via the Strategic Ledger.
*   **Knowledge Lineage:** Track how insights relate (SUPPORTS, CONTRADICTS, REFINES).
*   **Strategic Drift Detection:** AI-powered identification of contradictions in business strategy.
*   **Strategy Health Score:** Real-time dashboard indicator of strategic coherence and evidence grounding.

### 🧠 Knowledge Vault (Multi-Source RAG)
*   **Real-Time Ingestion Progress:** Visual feedback (percentage and status) during document chunking and embedding.
*   **Intelligence Ingestion:** Upload Pitch Decks, Financials, and Research.
*   **Worker-Powered Parsing:** Offloaded PDF/MD parsing and embedding generation using Piscina worker threads (Node.js 20-slim optimized).
*   **Semantic Memory:** Long-term storage of strategic insights with salience-based filtering.

### 📡 Strategic Intelligence Engine
*   **Intelligence Router:** Automatically determines if a request requires internal data or live web search.
*   **Real-Time Retrieval:** Integrated with Tavily API for current market facts, regulations, and competitor profiles.
*   **Source Attribution:** Mandatory clickable citations for all external data.

### 📈 Proactive Agency
*   **Daily Strategic Briefing:** AI-generated morning memos analyzing startup trajectory, risks, and today's top priority.
*   **Shareable Briefings:** One-click sharing of strategic insights via WhatsApp, Email, or Clipboard.
*   **BullMQ Pipeline:** Reliable background processing for complex strategic analysis.

### 🛡️ Hardened Security (BAM)
*   **Backend Authority Model:** The AI suggests tools; the Backend force-injects JWT-derived tenantId, preventing cross-tenant data leaks.
*   **Context Isolation:** Data strictly wrapped in XML tags to prevent prompt injection.
*   **Internal Data Hygiene:** AI is strictly forbidden from outputting internal database IDs (tenantId, userId).

## 🛠️ Technical Stack

### **Frontend**
*   **Framework:** Next.js 15 (App Router - Standalone Mode)
*   **UI:** Tailwind CSS + shadcn/ui + Framer Motion
*   **Charts:** Recharts (Theme-aware Execution Analytics)
*   **Streaming:** SSE (Server-Sent Events) for Chat and Vault Progress

### **Backend**
*   **Framework:** NestJS (Node.js 20-slim)
*   **ORM:** Prisma 7 (PostgreSQL + pgvector)
*   **Task Queue:** BullMQ (Redis)
*   **AI:** Google Gemini 1.5 Pro/Flash + @xenova/transformers (Local Embeddings)

## 🚦 Quick Start (Docker Preferred)

The entire MindAll stack is containerized for professional deployment.

### 1. Configure Environment
Rename `.env.example` to `.env` in the root directory and fill in your keys:
```env
# Database (Prisma 7 / pgvector)
DATABASE_URL="postgres://..."

# AI & Search
GEMINI_API_KEY="AIza..."
TAVILY_API_KEY="tvly-..."

# Security
JWT_SECRET="..."
JWT_REFRESH_SECRET="..."
```

### 2. Start the Stack
```bash
docker compose up --build -d
```

### 3. Verify
*   **Frontend**: `http://localhost:3000`
*   **Backend API**: `http://localhost:3001`
*   **Database**: `localhost:5432`
